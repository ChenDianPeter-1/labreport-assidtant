# Postlab Rules

Do not generate `reports/postlab_report.html` until:

- raw data are confirmed
- analysis is complete
- terminal step-by-step analysis review is approved
- final figures exist in `figures/`

The postlab must use:

- `source/` report requirements
- `analysis/raw-data/verified_raw_data.csv`
- `analysis/processed/processed_data.csv`
- `analysis/processed/summary_data.csv`
- `analysis/processed/calculation_chain.md`
- final figure files from `figures/`

Postlab structure constraints:

- Continue from prelab report structure baseline.
- Required section order:
  - 实验目的
  - 实验原理
  - 实验装置、设备及试剂
  - 实验步骤（预习准备工作）
  - 实验记录（实验过程中的第一手材料）
  - 数据处理与分析
  - 实验结论
  - 思考题
- Section `实验装置、设备及试剂` must not use tables.
- Section `实验记录` must be written after raw-data transcription confirmation
  and should reconstruct the actual lab process timeline.
- In `数据处理与分析`, first explain section organization and logic links, then
  present phased processing with input/operation/output/boundary.

Do not use placeholders for missing figures.
